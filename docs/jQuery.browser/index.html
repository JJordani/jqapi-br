jQuery.browser Retorna: Map

Descrição: Contém opções para o agente de usuário (UserAgent), lidas do navigator.userAgent. Nós desaconselhamos o uso dessa propriedade; por favor, tente usar a detecção de recurso ao invés (veja jQuery.support). O jQuery.browser talvez seja transferido para um plugin em uma futura versão do jQuery.

  • versão adicionada: 1.0 jQuery.browser

A propriedade $.browser fornece informação sobre o navegador que está acessando a página, conforme relatado pelo próprio navegador. Ele contém sinalizadores de cada uma das quatro categorias de navegador mais prevalentes (Internet Explorer, Mozilla, Webkit, e Opera), bem como informações sobre a versão.

Os sinalizadores disponíveis são:

  • webkit (a partir do jQuery 1.4)
  • safari (obsoleto)
  • opera
  • msie
  • mozilla

Esta propriedade está disponível imediatamente. É, portanto, seguro usá-la para determinar se deseja ou não chamar $(document).ready(). A propriedade $.browser é obsoleta no jQuery 1.3, e suas funcionalidades podem ser transferidas para um plugin de uma equipe de apoio em uma futura versão do jQuery.

Porque o $.browser utiliza o navigator.userAgent para determinar a plataforma, é vulnerável a spoofing pelo usuário ou deturpação do próprio navegador. É sempre melhor evitar código inteiramente específico de navegador sempre que possível. A propriedade $.support está disponível para detecção de suporte para características particulares ao invés de confiar em $.browser.

Exemplos:

Exemplo: Exibir informações do navegador. 

<!DOCTYPE html>
<html>
<head>
  <style>
  p { color:green; font-weight:bolder; margin:3px 0 0 10px; }
  div { color:blue; margin-left:20px; font-size:14px; }
  span { color:red; }
  </style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  
<p>Informações de navegador:</p>

<script>
    jQuery.each(jQuery.browser, function(i, val) {
      $("<div>" + i + " : <span>" + val + "</span>")
                .appendTo( document.body );
    });</script>

</body>
</html>

Demo:

Exemplo: Retorna true se o useragent é alguma versão do Microsoft Internet Explorer. 


  $.browser.msie;

Exemplo: Alerta "isso é WebKit!" somente para navegadores WebKit 


  if ($.browser.webkit) {
    alert( "isso é webkit!" );
  }

Exemplo: Alerta "Faça algo para o Firefox 3" apenas para navegadores Firefox 3. 


  var ua = $.browser;
  if ( ua.mozilla && ua.version.slice(0,3) == "1.9" ) {
    alert( "Faça algo para o firefox 3" );
  }

Exemplo: Defina uma propriedade CSS para um navegador específico. 


 if ( $.browser.msie ) {
    $("#div ul li").css( "display","inline" );
 } else {
    $("#div ul li").css( "display","inline-table" );
 }

jQuery.browser.version Retorna: String

Descrição: O número da versão do motor de renderização do navegador do usuário.

  • versão adicionada: 1.1.3 jQuery.browser.version

Aqui estão alguns resultados típicos:

  • Internet Explorer: 6.0, 7.0, 8.0
  • Mozilla/Firefox/Flock/Camino: 1.7.12, 1.8.1.3, 1.9
  • Opera: 10.06, 11.01
  • Safari/Webkit: 312.8, 418.9

Note que o IE8 afirma ser o 7 em modo de compatibilidade.

Exemplos:

Exemplo: Retora o número da versão do motor de renderização utilizado pelo navegador atual do usuário. Por exemplo, FireFox 4 retorna 2.0 (a versão do motor de renderização Gecko que é utilizada). 

<!DOCTYPE html>
<html>
<head>
  <style>
  p { color:blue; margin:20px; }
  span { color:red; }
  </style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  
<p></p>

<script>
$("p").html( "O número da versão do motor de renderização utilizado pelo seu navegador é: <span>" +
                $.browser.version + "</span>" );
</script>

</body>
</html>

Demo:

Exemplo: Alerta a versão do motor de renderização do IE que está sendo utilizado: 


if ( $.browser.msie ) {
  alert( $.browser.version );
}

Exemplo: Geralmente você só se preocupa com o "número principal", o número inteiro, que você pode recuperar usando a função nativa do JavaScript parseInt(): 


if ( $.browser.msie ) {
  alert( parseInt($.browser.version, 10) );
}
